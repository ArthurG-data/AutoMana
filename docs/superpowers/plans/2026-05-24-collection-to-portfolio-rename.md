# Collection → Portfolio Rename Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Rename the `/collection` route to `/portfolio`, update all user-facing labels from "Collection" to "Portfolio", and wire the sidebar to the new route.

**Architecture:** This is a pure frontend rename. The backend API, feature folder (`features/collection/`), TypeScript types, and query keys all stay as-is — "collection" is the data-domain name and does not change. Only the route path, route file, CSS module, and UI labels change. The `routeTree.gen.ts` is auto-generated by TanStack Router's vite plugin; we update it manually here since we're not running the dev server.

**Tech Stack:** React 18, TanStack Router (file-based routing), TanStack Query, CSS Modules, Vitest + Testing Library
