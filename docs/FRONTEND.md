# automana Frontend

The automana frontend is a React 18 SPA located at `src/frontend/`. It runs independently of the FastAPI backend: in development all API calls are intercepted by MSW so no running backend is needed.

---

## Tech Stack

| Layer | Library | Version |
|---|---|---|
| Build | Vite | 5.x |
| UI | React | 18.3 |
| Language | TypeScript | 5.5 |
| Routing | TanStack Router | 1.x (file-based) |
| Server state | TanStack Query | 5.x |
| Client state | Zustand | 5.x |

## Quick Start

```bash
cd src/frontend
npm install
npm run dev          # http://localhost:5173 ŌĆö MSW active, no backend needed
```

Other scripts:

```bash
npm test             # unit + integration (Vitest, single run)
npm run test:watch   # Vitest watch mode
npm run test:ui      # Vitest browser UI
npm run test:e2e     # Playwright (installs Chrome on first run: npx playwright install chromium)
npm run build        # TypeScript check + Vite production build
npm run preview      # serve the production build locally
```

The dev server proxies `/api/*` to `http://localhost:8000` (FastAPI). In dev mode, MSW intercepts before the proxy so the backend is never hit.

## Theming

All colours, shadows, and fonts are CSS custom properties defined in `src/styles/tokens.css`. Two themes ship:

| Theme | Name | Trigger |
|---|---|---|
| Dark (default) | Deep Sea | `:root` |
| Light | Arctic | `[data-theme='light']` on `<html>` |

`useUIStore.setTheme('light')` writes `document.documentElement.dataset.theme = 'light'`; removing the attribute restores dark. The preference is persisted to `localStorage` under the key `automana-ui` and re-applied on page load via Zustand's `onRehydrateStorage` hook.

### Token reference

```
--hd-bg           page background
--hd-surface      card / panel background
--hd-surface-alt  secondary surface (hover states, track fills)
--hd-border       default border (low contrast)
--hd-border-hot   hover / focus border (higher contrast)
--hd-text         primary text
--hd-muted        secondary text
--hd-sub          tertiary text / labels
--hd-accent       primary action colour (teal dark / green light)
--hd-accent-rgb   RGB triplet for accent (used in rgba() / color-mix())
--hd-blue         informational / link colour
--hd-blue-rgb     RGB triplet for blue
--hd-red          destructive / price-down colour
--hd-amber        warning / price-up colour
```

## State Management

### Auth store ŌĆö `src/store/auth.ts`

Persisted to `localStorage` under key `automana-auth`.

```ts
import { useAuthStore } from '../store/auth'

const token = useAuthStore.getState().token     // outside React
const { login, logout } = useAuthStore()        // inside React
```

**State shape:**

```ts
{
  token: string | null        // JWT or dev-stub-token
  currentUser: { id: string; email: string } | null
  login: (token, user) => void
  logout: () => void
}
```

**Dev stub:** The store initialises with `token: 'dev-stub-token'` and `currentUser: { id: 'dev', email: 'dev@automana.local' }`. All authenticated routes are accessible without logging in during development.

When the real auth endpoint ships, replace the initial state with `token: null, currentUser: null`.

### UI store ŌĆö `src/store/ui.ts`

Persisted to `localStorage` under key `automana-ui`.

```ts
import { useUIStore } from '../store/ui'

const { theme, setTheme, toggleTheme } = useUIStore()
```

**State shape:**

```ts
{
  theme: 'dark' | 'light'
  setTheme: (theme) => void
  toggleTheme: () => void
}
```

Both `setTheme` and `toggleTheme` call `applyTheme` internally, which sets/clears `document.documentElement.dataset.theme`.

## API Layer

### `apiClient` ŌĆö `src/lib/apiClient.ts`

Thin `fetch` wrapper. Reads the bearer token from the auth store on every call.

```ts
import { apiClient, ApiError } from '../lib/apiClient'

const data = await apiClient<CardSearchResponse>('/cards/search?q=ragavan')
```

- Prefixes all paths with `/api/v1`
- Sets `Content-Type: application/json`
- Injects `Authorization: Bearer <token>` when a token exists
- Throws `ApiError` (with `.status: number`) on any non-2xx response

Callers should never catch 401 themselves ŌĆö the `queryClient` handles it globally.

### `queryClient` ŌĆö `src/lib/queryClient.ts`

Singleton `QueryClient` shared across the app via `<QueryClientProvider>` in `main.tsx`.

**Configuration:**

| Setting | Value |
|---|---|
| `staleTime` | 30 000 ms |
| Retry | up to 2 times, never retries 401 |
| 401 handler | calls `useAuthStore.getState().logout()` on any query or mutation error with status 401 |

## Routing

Routes use TanStack Router's file-based convention. The route tree is **auto-generated** at `src/routeTree.gen.ts` by the Vite plugin when the dev server starts. Never edit this file manually.

### Route map

| File | Path | Auth required |
|---|---|---|
| `__root.tsx` | (root) | ŌĆö |
| `index.tsx` | `/` | No |
| `login.tsx` | `/login` | No |
| `search.tsx` | `/search` | Yes |
| `cards.$id.tsx` | `/cards/:id` | Yes |

### Auth guard

`__root.tsx` runs a `beforeLoad` hook on every navigation:

```ts
const PUBLIC_PATHS = ['/', '/login']

beforeLoad: ({ location }) => {
  const token = useAuthStore.getState().token
  if (!token && !PUBLIC_PATHS.includes(location.pathname)) {
    throw redirect({ to: '/login' })
  }
}
```

### Router context

The `QueryClient` is passed as router context so route loaders can prefetch data:

```ts
// main.tsx
const router = createRouter({ routeTree, context: { queryClient } })

// any route
loader: ({ context: { queryClient } }) =>
  queryClient.ensureQueryData(cardSearchQueryOptions({}))
```

### Search param validation

`/search` validates its URL params with Zod via `validateSearch`:

```ts
const searchSchema = z.object({
  q:        z.string().optional(),
  rarity:   z.string().optional(),
  finish:   z.string().optional(),
  minPrice: z.number().optional(),
  maxPrice: z.number().optional(),
})
```

TanStack Router serialises/deserialises these automatically and provides them typed to `Route.useSearch()`.

### Adding a new page

1. Create `src/routes/<name>.tsx` (use `$param` in the filename for dynamic segments).
2. Export `Route = createFileRoute('/<name>')({ component, loader?, validateSearch? })`.
3. Restart the dev server ŌĆö `routeTree.gen.ts` regenerates automatically.
4. If the page is authenticated, no extra work needed; the root `beforeLoad` guard covers all non-public paths.

## MSW Mock Layer

MSW runs as a Service Worker in the browser (dev only) and as a Node server in tests.

| File | Purpose |
|---|---|
| `mocks/data.ts` | 8 `CardSummary` fixtures + 1 full `CardDetail` for `ragavan-mh2` |
| `mocks/handlers.ts` | Handles `GET /api/v1/cards/search` (q/rarity/finish filtering) and `GET /api/v1/cards/:id` (404 on unknown id) |
| `mocks/browser.ts` | `setupWorker(...handlers)` ŌĆö imported lazily by `main.tsx` in DEV |
| `mocks/server.ts` | `setupServer(...handlers)` ŌĆö imported by `test-setup.ts` in tests |

MSW is **not** included in production builds. The `if (import.meta.env.DEV)` guard in `main.tsx` tree-shakes it out.

### Adding a new mock endpoint

```ts
// src/mocks/handlers.ts
import { http, HttpResponse } from 'msw'

http.get('/api/v1/portfolio/summary', () =>
  HttpResponse.json({ total_value: 12400, card_count: 340 })
)
```

Add the handler to the `handlers` array. Both browser worker and test server pick it up automatically.

## Development Conventions

**CSS Modules:** every component that needs scoped styles gets a colocated `.module.css` file. Global/shared rules go in `src/styles/global.css`. Use `var(--hd-*)` tokens everywhere ŌĆö never hardcode colours.

**No barrel imports for `features/`:** import directly from `../features/cards/api` rather than a hypothetical `features/index.ts`. Barrel files are only used for `components/design-system/index.ts` and `components/ui/index.ts`.

**Query key discipline:** query keys live in `features/*/api.ts` alongside their `queryFn`. Never construct a raw array key at the call site.

**Route loaders prefetch:** always call `queryClient.ensureQueryData(...)` in the route loader so data is available before the component mounts. This eliminates loading spinners on initial navigation.

**`useSuspenseQuery` inside components:** pair with the loader above. The component can destructure `data` directly ŌĆö it is guaranteed to be defined.

## Connecting to a Live Backend

Phase 1 runs fully on MSW. To switch to the real FastAPI backend:

1. Start the backend: `dcdev-automana up -d` (see `docs/DEPLOYMENT.md`)
2. Remove (or comment out) the `enableMocking()` call in `src/main.tsx`, or set `import.meta.env.DEV` to false
3. Replace the dev-stub initial state in `src/store/auth.ts` with `token: null, currentUser: null`
4. Implement the `/api/v1/auth/login` endpoint and wire `useAuthStore.login()` to its response

The `apiClient` and `queryClient` are already production-ready ŌĆö no changes needed there.
